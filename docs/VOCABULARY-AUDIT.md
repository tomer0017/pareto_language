# READY — Situation Vocabulary Audit & Priming Strategy

Status: **living document.** Covers the "Core Words / Core Sentences / Review / Mission Priming"
sprint. Honest about what is implemented now vs. what is audited-but-deferred.

READY's rule is unchanged: **sentences are the learning unit; words exist only to make sentences
easier to understand, remember, and use.** Vocabulary priming is *preparation*, never the destination.

---

## 1. Where vocabulary lives (three layers, no duplication)

| Layer | Source of truth | Surface |
|---|---|---|
| **Global Core Words** | `content/core-corpus/data/*.ts` → pipeline → `public/content/core-{lang}.v1.json` | Core → Words (Browse · Picture Quiz · Swipe Recall) |
| **Core Sentences** | the mission `items` in `apps/web/src/features/bootcamp/**` | Core → Phrases + **Sentence Flashcards** |
| **Mission priming** | `{ kind: 'prime' }` steps inside a mission | shown in the mission flow, right before the sentence it prepares |

Sentence flashcards reference canonical mission item ids (`buildSentenceDeck` in
`apps/web/src/features/core/flashcards.ts`) — they are **not** a hand-maintained copy. Priming words
are new small atoms (e.g. `milk`, `avec`), and the *assembled sentence* references a canonical
mission item via `prime.buildFromItemId` — so nothing is duplicated.

## 3. Numbers — audit result

`content/core-corpus/data/numbers-time.ts` stores only the **non-compositional atoms**
(`one`–`twelve`, `twenty, fifty, hundred, thousand, half`) and drives compositional practice through
generators (methodology §4 "Never-Teach": 21–99 are drilled, never stored as 79 separate cards).
This is the pattern-based approach Part 8 asks for.

- **English** ✅: 0–12 + tens + hundred/thousand atoms; combinations (`twenty-one`) are generated.
- **French** ✅ **now implemented**: `apps/web/src/features/bootcamp/fr/frenchNumbers.ts` is the ONE
  tested source of truth for spoken `fr-FR` numbers 0–9999, encoding the vigesimal rules:
  `soixante-dix (70)`, `soixante et onze (71)`, `quatre-vingts (80, final -s)`, `quatre-vingt-un (81,
  no -s / no "et")`, `quatre-vingt-dix (90)`, `quatre-vingt-onze (91)`, `quatre-vingt-dix-neuf (99)`,
  `cent (100)`, `deux cents` / `deux cent un`, `mille (1000)`. Pinned by `frenchNumbers.test.ts` (36
  cases). Exposed as `FR_TRICKY_NUMBERS` + `FR_PRICE_EXAMPLES` (7/15/20/25/70/80/90/95 €) and taught
  in-situation via a **French-numbers priming step in Mission 3** whose word forms are generated by
  `frenchNumber()` (mission and module can never drift). Still AI-drafted / pending native review.

## 4. Complete mission-by-mission audit (all 30)

The machine-checkable version is `apps/web/src/features/bootcamp/vocabAudit.ts`
(`MISSION_VOCAB_AUDIT`), bound to reality by `vocabAudit.test.ts`: the DECISION for every mission
must match whether the mission actually has a `prime` step, and every listed priming word must be a
real prime word in that mission. **Counts: 30 audited · 8 primed · 22 explicitly no-priming-needed.**

**Primed missions (8) — a zero-level learner meets new building blocks:**

| # | Situation | New priming words | Reviews | Builds sentence |
|---|---|---|---|---|
| 1 | Recovery | sorry, I, understand, slowly, please | — | "Sorry, I don't understand." |
| 2 | Introduce Myself | name, from, holiday, first time | — | "I'm here on holiday." |
| 3 | Numbers & Money | how much, euros, cash, card, five, ten | — | "How much is it?" |
| 4 | Café | coffee, milk, sugar, medium, with, no/without | — | "Milk, no sugar." |
| 5 | Restaurant | table, menu, water, bill | **please** | "The bill, please." |
| 6 | Directions | excuse me, left, right, straight, near, far | — | "Turn left…" |
| 7 | Taxi | address, airport, stop, here | **how much** | "To this address, please." |
| 8 | Hotel Check-in | reservation, night, breakfast, passport | **name** | "I have a reservation." |

French parity: FR Missions 1–4 (all currently built) carry the same priming; FR Mission 3 primes the
**vigesimal number patterns** (`cinq, dix, vingt, soixante-dix, quatre-vingts, quatre-vingt-dix`).

**No-priming-needed missions (22)** — each has an explicit recorded reason. Summary of the pattern:
- **Reuse** (9, 13, 14, 15, 16, 17, 19, 20, 21, 22, 23, 25, 26): the situation's building blocks were
  already introduced earlier (sizes, with/without, how-much, recovery tools) — now global Core, so no
  re-teaching. Example: Mission 13 (Restaurant Basics) reuses the exact nouns primed in Mission 5.
- **Checkpoints / cold missions** (10, 18, 24, 28, 29, 30): concepts target = 0 by design — cold
  integration, no new content, so priming would violate the format.
- **Overlearned / safety-read** (27 Emergency; 11 Border; 12 Hotel Requests): emergency phrases must
  be automatic not freshly primed; allergen/drug/document names are *read on a card*, not drilled by
  ear (RoF-3 safety — Part 11).

The "new vs review" honesty is enforced: a word may only carry `review: true` if it was introduced in
an earlier mission (`priorPrimeVocabulary` in `primeVocab.ts` + `prime.test.ts`).

### Prior-knowledge model
`primeVocab.ts` walks missions in plan order and accumulates the set of words introduced as *new*
priming. Later missions mark a reused word `review: true` (rendered with a ♻️ hint) instead of
re-teaching it. Tests assert every review word really did appear earlier, and every "new" word really
is new — so coverage can grow without silently duplicating vocabulary.

## 5. How to add priming to a new mission

1. In the mission's `steps`, insert before the target sentence's `tool` step:
   ```ts
   { kind: 'prime', label: T('לפני שנדבר', 'Before we speak'),
     intro: T('…', '…'),
     words: [ { text: 'milk', meaning: T('חלב', 'milk'), emoji: '🥛' }, /* 3–6, up to 8 */ ],
     buildFromItemId: 'en.phrase.coffee.no-sugar' },
   ```
2. Keep it to **3–6 words** (8 max — enforced by `prime.test.ts`).
3. `buildFromItemId` must be a real mission item; the test also checks the prime appears *before* that
   sentence's tool step and that a primed word is actually part of the sentence.
4. No code changes — the `PrimeStep` renderer and the language-agnostic registry already handle it.

## 6. Guardrails honored (Part 11)

Sentences stay primary · dialogues stay the context · no standalone vocab lists before *every*
mission · no grammar lectures · no conjugation tables · narrative dialogue order is never randomized
(only answer options / review order are) · every primed word answers "does this help a near-beginner
handle a real travel moment sooner?".
